Building Tatool Web

Tatool Web uses gulp for its build process automation. Details about the tasks (concat, minify, copy) in the build process can be found in gulpfile.js.

$> gulp build

Run Tatool in lab mode, which deactivates the registration and email features and adds a default admin user.

$> npm start

# alternatively you can call node directly
$> node server.js lab dev

If you want to run Tatool Web on a webserver or in production mode, you'll have to provide the following two configuration parameters. The environment parameter will direct Tatool Web to either access your development version (app directory) or the built version (dist directory). In order to use the built version you must run npm install first.

# [mode] : lab / server
# [environment] : dev / prod
$> node server.js [mode] [environment]
